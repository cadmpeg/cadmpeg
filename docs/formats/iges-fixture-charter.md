# IGES fixture charter

## Original fixtures

Original fixtures are generated from explicit builders whose inputs are field values, entity definitions, and line-ending choices. Builders serialize the rules in `iges.md`; they do not ingest, rewrite, minimize, or transform external files. Each fixture declaration identifies its envelope, entity types and forms, malformed condition if any, and asserted ladder gates.

The original matrix covers:

- canonical and noncanonical physical lines, all section transitions, count mismatches, and post-Terminate bytes;
- default and alternate delimiters, Hollerith values crossing cards, exponent spellings, omitted values, continuation cards, and trailing associativity and property groups;
- Directory Entry pairing, back-pointers, pointer parity, dangling references, wrong target types, and permitted and forbidden cycles;
- every entity/form row in `corpus/iges-envelope-a.toml`, including default and non-default branches;
- unit systems, model scale, nested transforms, reflection, and definition/occurrence ownership;
- face-local trimming, explicit shared B-rep, reversed uses, seams, degeneracy, void shells, and non-manifold rings;
- presentation, product, external-reference, annotation, associativity, property, view, and drawing graphs;
- exact byte-ledger coverage for canonical, short, extended, alternate-ending, malformed, opaque, and post-Terminate records.

Generated fuzz seeds are selected original fixtures with bounded size. Mutation tests operate on generated bytes and retain the generator declaration that establishes the expected invariant.

## Public fixtures

Public fixtures enter through the corpus manifest only when their redistribution terms, source URL, acquisition date, checksum, and applicable envelope are recorded. Public proof classes cover mainstream mechanical files, explicit B-rep and trimmed-sheet topology, product structure, external references, appearance, annotations, and drawings. A public fixture supports a score only when deterministic reports show its entity/forms, decoded domains, losses, topology validation, and byte coverage.

`corpus/iges-public-evidence.toml` maps an accepted IGES manifest filename to the fixture classes, assertions, and exact codec tests it exercises. It contains no duplicate provenance metadata. `iges-proof-report` verifies each mapping against `corpus/manifest.toml`, the committed bytes, the manifest SHA-256, and the named test functions. A fixture class counts for a matrix row or ladder gate only when the relevant fixtures collectively name every assertion required by that row or gate. Unknown, duplicate, or missing fixture classes, assertions, or tests invalidate the proof.

`corpus/iges-original-evidence.toml` maps every original fixture class to exact codec test names. The generator verifies that the tests exist and that every matrix decoder path resolves to an owning source function. Generate or verify the deterministic cumulative report with:

```sh
cargo run -p cadmpeg-codec-iges --bin iges-proof-report -- --write corpus/iges-envelope-a-proof.json
cargo run -p cadmpeg-codec-iges --bin iges-proof-report -- --check corpus/iges-envelope-a-proof.json
```

`corpus/iges-ladder-gates.toml` maps every cumulative L0 through L8 gate to fixture classes, assertions, and required format decisions. `corpus/iges-ladder-decisions.toml` records the settled L0, L3, L4, L6, and L7-mates format decisions. A decision-dependent gate remains incomplete until `corpus/iges-approvals.toml` records maintainer approval. That file also records approval of the closed matrix and byte-ledger policy. The report sets `release_ready` only when all three approvals are present, every cumulative gate is complete, and every admitted row has a real decoder, destination, assertion, original fixture class, and verified public fixture class.

## Machine-checkable gates

- L0: bounded detection, document kind, Fixed ASCII framing, version, and Global metadata assertions.
- L1: section navigation, entity census, external references, named undecoded content, reference findings, and full physical byte coverage.
- L2: typed point, analytic, spline, NURBS, and procedural-carrier evaluation with units and placement.
- L3: validated body-to-vertex explicit topology and validated face-local sheet topology without inferred adjacency.
- L4: the accepted inapplicability decision is present in the generated profile.
- L5: every admitted geometry/topology matrix row has a passing decoder, original fixture, public fixture class, carrier assertion, and appearance assertion.
- L6: the accepted inapplicability decision is present in the generated profile.
- L7: typed definition, occurrence, group, placement, external-reference, and persistent-identity graphs pass; the accepted mates inapplicability decision is present.
- L8: every admitted presentation and application row is typed or deliberately retained with identity, every relationship validates, and each fixture ledger covers exactly the source length with zero gaps and overlaps.
